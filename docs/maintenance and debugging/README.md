# Maintenance and Debugging

This folder collects operational notes, debugging history, and maintenance procedures for the AI Debate Platform.

## Contents
- `runbook.md` - Common commands, server lifecycle, and provider test steps.
- `debugging-log.md` - Issues found during development and how they were resolved.
- `maintenance-todo.md` - Future cleanup and reliability tasks.

## Ground Rules
- Never document real API keys.
- Keep provider errors redacted before sharing logs.
- Run tests and Ruff before pushing changes.
- Keep code files under the submission guideline limit of 150 lines.
